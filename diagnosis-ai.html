Commerce AI Diagnosis Platform

A FastAPI-based Enterprise AI diagnosis platform that compares product records between Stage and Production environments. The current implementation uses LangGraph orchestration, MCP-connected tools, domain-specific diagnosis logic, root-cause aggregation and local LLM summary generation with Cloud LLM fallback. The architecture is designed to evolve into a future multi-agent system with Supervisor, Diagnosis and Summary Agents.


FastAPI LangGraph Orchestration MCP Tools Root Cause Analysis Local LLM Groq Fallback Future Multi-Agent Architecture

Problem

Commerce product behavior can differ between Stage and Production because of catalog, inventory, pricing, publishing or product-state differences. Business users and customer service teams need a faster and more consistent way to identify root causes.

Current Solution

The platform accepts a part number, fetches Stage and Production records through MCP-connected tools, runs domain-specific diagnosis logic and returns root cause, severity, recommendations and an executive summary.

Business Value

Reduces manual investigation, improves diagnosis accuracy, standardizes support workflows and helps business and support teams resolve commerce issues faster.

Diagnosis Components

  • Catalog Diagnosis: buyable, available, published and mark-for-delete checks
  • Inventory Diagnosis: Stage vs Production inventory differences
  • Pricing Diagnosis: missing or mismatched price diagnosis
  • LangGraph Workflow: orchestrates the diagnosis process
  • MCP Tools: retrieve Stage and Production product data
  • Summary Logic: generates executive summary with fallback support

Future Multi-Agent Enhancement

The next enhancement is to evolve this LangGraph workflow into a Supervisor-driven Multi-Agent Architecture. The Supervisor Agent will route requests to specialized Diagnosis Agents and Summary Agents, which will call MCP-connected tools to retrieve enterprise data and generate structured recommendations.

Sample Diagnosis Rules

  • Stage buyable but Production not buyable
  • Stage published but Production unpublished
  • Stage or Production product marked for delete
  • Inventory exists in Stage but missing in Production
  • Price exists in Stage but missing in Production
  • Stage and Production product-state mismatch

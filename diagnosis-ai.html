Diagnosis rules · 03

Evidence before explanation.

The system applies deterministic domain checks first. The model then summarizes structured findings rather than inventing the diagnosis.

Catalog

Product state checks

  • Buyable and available flags
  • Published versus unpublished state
  • Mark-for-delete lifecycle status
Inventory

Stock comparison

  • Stage inventory exists but Production is missing
  • Quantity and availability mismatches
  • Environment-specific stock evidence
Pricing

Commercial data checks

  • Missing Production price
  • Stage and Production value mismatch
  • Pricing evidence attached to findings
Planned enhancement · development not started

Supervisor-driven multi-agent orchestration

The proposed next phase introduces a Supervisor routing requests to Catalog, Inventory, Pricing, and Summary agents. This is an architecture plan for future development, not a capability in the current repository.

Resilience

Summary fallback

Local generation remains the primary path, with cloud model fallback available for resilient executive summaries.

Engineering learnings

What the diagnosis workflow made clear

  • LangGraph simplifies orchestration only when state transitions remain explicit
  • MCP separates reasoning logic from product-data access and tool contracts
  • Deterministic domain checks should run before LLM summarization
  • Root-cause quality is limited by the completeness of the collected evidence
